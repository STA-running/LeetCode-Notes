## 一、刷题语言选择

**用 C++ 刷，不用 Java。**

原因很直接：你的求职目标是 Qt/C++ 岗，面试现场大概率要手写 C++，用 Java 刷题等于练了副业。你有 ACM 竞赛基础，C++ 语法不陌生，直接上。

```cpp
// 力扣 C++ 模板写法示例（Two Sum）
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        unordered_map<int, int> map;
        for (int i = 0; i < nums.size(); i++) {
            int comp = target - nums[i];
            if (map.count(comp)) return {map[comp], i};
            map[nums[i]] = i;
        }
        return {};
    }
};
```

## 三、核心原则

1. **一题多解**：先想暴力，再想优化，写出来后看题解对比，搞懂最优解的思维。同类型的题集中练（连刷 3 道链表），形成肌肉记忆。
2. **错题必复刷**：做错或想了超过 15 分钟没思路的题，记下来，3 天后重做一遍。
3. **不做 Hard**：工控/制造业中小公司面试，Hard 题几乎不考，Medium 做到顺滑就够。
4. **口述每道题**：做完一题，摘掉耳机，自己说一遍思路——不是默念，是真说出口——面试官要听你讲。
5. **不追求速度追求理解**：一天 1 道吃透比一天 3 道囫囵吞枣强。

## 八、常用刷题技巧速记

| 技巧          | 场景                      | 记忆口诀                                          |
| ------------- | ------------------------- | ------------------------------------------------- |
| 快慢指针      | 链表判环/找中点/删倒数第N | 快走两步慢一步，相遇即有环                        |
| 左右双指针    | 有序数组求和/接雨水/盛水  | 先排序再夹逼，哪边小移哪边                        |
| 前缀和 + 哈希 | 和为K的子数组             | `sum[i] - sum[j-1] = K` → `sum[j-1] = sum[i] - K` |
| 单调栈        | 下一��更大/每日温度       | 栈顶比当前小就弹出，大的留着                      |
| BFS 网格模板  | 岛屿/腐烂橘子             | 队列存坐标，四个方向扩散，记录步数                |
| DP 递推       | 爬楼梯/打家劫舍           | 先写暴力递归 → 加 memo → 推 dp 数组               |
| 回溯模板      | 全排列/子集/组合          | `for(选择){做选择;递归;撤销选择}`                 |
