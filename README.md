# LeetCode 刷题笔记 · 56 题精刷实录

这不是一份普通的题解集。这是一份**从零到一的刷题思考全过程记录**。

## 这份笔记有什么不同？

市面上大部分刷题笔记只会告诉你"最终解法是什么"，但**不会告诉你为什么这么想、踩过哪些坑、卡在哪里、最后怎么突破的**。

这份笔记记录了每一个关键问题的思考过程：

- **为什么哈希表查找是 O(1) 而不是 O(n)？** —— 哈希值计算 → 直接定位到桶，不是逐个比较
- **为什么 `ans += map[target]` 而不是 `ans++`？** —— 同一个前缀和可能出现多次，每次对应一个不同的合法子数组
- **为什么 while 循环不是 O(n²)？** —— 摊还分析：每个元素最多 swap 一次，总次数 ≤ n
- **为什么不能用 if 代替 while？** —— if 只管一次 swap，swap 过来的新元素可能仍不在正确位置
- **没学过"原地哈希"能写出来吗？** —— 能。从哈希表原理出发，自己定义 `hash(k) = k-1`，把数组本身当哈希表用

**这些不是"标准答案"，而是真实的思考轨迹。**

## 这份笔记适合谁？

- **刚起步的刷题者**：能看到一个新手如何从零理解每道题，而不是直接看最优解
- **卡在某个知识点的学习者**：每个知识点的总结都来自真实踩坑后的理解
- **准备面试的求职者**：题解中包含"面试官可能会追问什么"的深度思考

## 核心方法论

### 推导 > 背诵

不追求"见过这道题所以会做"，而是追求"没见过也能推出来"。比如：

- 双指针三连（26/27/283）：三道题共用一个模板，区别只在**保留条件**这一行
- 和为 K 的子数组：从暴力 O(n²) 出发，发现瓶颈是"往回查"，用哈希表突破到 O(n)
- 缺失的第一个正数：从"不能用哈希表"的约束出发，自己定义了哈希函数

### 精度 > 数量

> 56 题精刷，质量优先。不是冲 200 道一知半解，而是每道都搞透。

- 宁可花 15 分钟 debug 一个 while/if 的区别，也不赶着刷下一道
- 每道题先自己想 15 分钟，再看题解对比思路差异
- 做完后整理成笔记，把"为什么这么想"写清楚

### 模板思维

第一阶段刷下来，发现很多题是**同一个妈生的**：

- **哈希表模板**：两数之和 → 和为 K 的子数组（同一套查表逻辑）
- **双指针模板**：26/27/283 → 改一个条件就换一道题
- **转置 + 反转**：旋转 90° 的通用解法
- **原地哈希**：数组本身当哈希表用，打破"只能用 STL"的思维定势
