# 双指针技巧

双指针是数组题最常用的技巧之一。核心思想是：**用两个指针以不同的速度或方向遍历数组，完成单指针需要嵌套循环才能做的事**。

## 两种模式

### 模式 A：同向而行（快慢指针）

两个指针从同一端出发，速度不同。

```
慢指针 k：指向"下一个写入位置"，只在写入时前进
快指针 i：扫描全数组，每次循环都前进
```

**通用模板：**

```cpp
int k = 开始位置;
for (int i = 开始位置; i < n; i++) {
    if (满足保留条件) {
        nums[k] = nums[i];    // 或 swap(nums[k], nums[i])
        k++;
    }
    // 不满足 → 跳过
}
return k;
```

**应用：**

| 题号 | 保留条件 | k 初始值 |
|:----:|---------|:--------:|
| 26 删除有序数组中的重复项 | `nums[i] != nums[k-1]` | 1 |
| 27 移除元素 | `nums[i] != val` | 0 |
| 283 移动零 | `nums[i] != 0` | 0 |

**关键理解——不变量：**

`nums[0..k-1]` 永远是已经确认要保留的正确数据，这条分界线在算法运行期间始终成立。k 只能通过 `k++` 增加，不能随意赋值。

### 模式 B：相向而行（左右夹逼）

两个指针分别从两端出发往中间靠拢。

```
left 指向最左端（最小值），right 指向最右端（最大值）
根据当前结果决定移动 left 还是 right
```

**应用场景：**

| 题目 | 移动依据 |
|------|---------|
| 三数之和 | 根据 `nums[i]+nums[l]+nums[r]` 与 target 的大小 |
| 盛最多水的容器 | 哪边柱子矮就移哪边 |

## 快慢指针 vs 左右夹逼

| 区别 | 快慢指针 | 左右夹逼 |
|------|---------|---------|
| 方向 | 同向 | 相向 |
| 依赖排序 | 不依赖 | 通常需要排序 |
| 典型用途 | 删除/移动/覆盖 | 求和/求面积/搜索 |
| 复杂度 | O(n) | O(n) |
