# 哈希表与哈希集合

## 核心思想

哈希表（`unordered_map`）和哈希集合（`unordered_set`）能在 **平均 O(1)** 时间内完成元素的查找、插入和删除。

> **本质**：把要查的值 x，通过哈希函数映射到一个位置 `hash(x)`，存到那里。下次查 x 时直接去那个位置拿。

## 关键区别

| 数据结构 | 存什么 | 典型用途 |
|----------|--------|---------|
| `unordered_set` | 单个值 | "这个数出现过没有？" |
| `unordered_map` | 键值对 (key, value) | "这个数出现了几次？" / "这个数对应的下标是什么？" |

## 刷题中的经典用法

### 1. 两数之和（模板题）

```cpp
unordered_map<int, int> hash;   // value → index
for (int i = 0; i < n; i++) {
    int complement = target - nums[i];
    if (hash.count(complement)) return {hash[complement], i};
    hash[nums[i]] = i;
}
```

**模式**：遍历时查"我要的另一个数之前出现过吗？"

### 2. 存在重复元素

```cpp
unordered_set<int> seen;
for (int num : nums) {
    if (seen.count(num)) return true;
    seen.insert(num);
}
return false;
```

**模式**：边走边查，见过就说明重复了。

### 3. 两个数组的交集

```cpp
unordered_set<int> s(nums1.begin(), nums1.end());
vector<int> res;
for (int num : nums2) {
    if (s.count(num)) {
        res.push_back(num);
        s.erase(num);        // 防止重复
    }
}
```

**关键技巧**：找到一个交集元素后 `erase` 掉，一行代码搞定去重。

### 4. 和为 K 的子数组

```cpp
unordered_map<int, int> map;   // prefix_sum → count
map[0] = 1;
int cur = 0, ans = 0;
for (int num : nums) {
    cur += num;
    int target = cur - k;
    if (map.count(target)) ans += map[target];   // 不是 ans++！
    map[cur]++;
}
```

**核心理解**：`ans += map[target]` 而不是 `ans++`，因为同一个前缀和可能出现多次，每次出现都能构成一个新的合法子数组。

## 常见陷阱

1. **`find` 返回迭代器，不能当布尔用**
   ```cpp
   if (hash.find(x))          // ❌ 编译错误
   if (hash.find(x) != hash.end())  // ✅
   ```

2. **`unordered_map::insert` 的参数是 pair**
   ```cpp
   map.insert(0, 1);          // ❌
   map[0] = 1;                // ✅
   map.insert({0, 1});        // ✅
   ```

3. **重复元素导致性能问题**：遍历 `nums` 而不是 `set` 时，相同值的元素会触发重复查询，最坏 O(n²)。解决办法：遍历 `set`。

## 什么时候用数组替代哈希表？

当数据范围已知且很小时，用数组比哈希表快几十倍：

```cpp
// 字母异位词：只有 26 个小写字母
int count[26] = {0};
count[c - 'a']++;     // 直接寻址，一条指令
```

> **栈上开数组 ≈ 一条指令，哈希表 ≈ 几十条指令。** 能用数组就不要用哈希表。
