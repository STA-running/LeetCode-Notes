# 原地哈希技巧

## 什么时候需要原地哈希？

当题目要求 **O(1) 额外空间** 时，不能使用额外的 `unordered_set` 或 `unordered_map`。但可以利用**数组本身**来充当哈希表。

## 核心思想

> 把数组本身当哈希表用，自己定义哈希函数。

### 实践：缺失的第一个正数

**题目**：找出未排序数组中没有出现的最小正整数。要求 O(n) 时间，O(1) 额外空间。

**推理过程**：

1. 答案一定在 [1, n+1] 范围内
2. 所以只关心 1 ~ n 这些数
3. 定义哈希函数：`hash(k) = k - 1`（数字 k 应该去下标 k-1）
4. 把每个在范围内的数 swap 到正确位置
5. 遍历数组，第一个 `nums[i] != i+1` 的位置就是答案

```cpp
int n = nums.size();
for (int i = 0; i < n; i++) {
    while (nums[i] > 0 && nums[i] <= n && nums[i] != nums[nums[i] - 1]) {
        swap(nums[i], nums[nums[i] - 1]);
    }
}
for (int i = 0; i < n; i++) {
    if (nums[i] != i + 1) return i + 1;
}
return n + 1;
```

## while vs if 的本质区别

| 写法 | 行为 | 结果 |
|------|------|------|
| `if` | swap 一次就走 | swap 过来的新元素可能仍需处理，被晾在那 |
| `while` | 持续 swap 直到当前位置没法再处理 | 万无一失 |

## 复杂度分析

这是一个**摊还分析**的经典例子。看起来是二重循环：

```cpp
for (int i = 0; i < n; i++) {
    while (条件) { swap(...); }
}
```

但每次 swap 都把**至少一个元素送到正确位置**，而一个元素最多被正确放一次。所以所有 while 的 swap 总次数 ≤ n，整体 O(n)。

## 这种排序为什么能 O(n)？

教科书说的"排序最低 O(n log n)"是指**比较排序**。本题的排序**不靠比较**，而是利用了额外信息：**值范围是 [1, n] + 每个数知道该放哪**。非比较排序可以突破 O(n log n) 的下限。

## 什么时候用这个技巧？

当题目满足以下条件时可以考虑原地哈希：
- 值范围有限（如 [1, n]）
- 要求 O(1) 额外空间
- 需要快速判断某个值是否存在
