# 前缀和与前缀积

## 核心思想

前缀和/积允许用 **O(1) 时间** 计算任意子数组的连续和/积。

### 定义

```
nums   = [a0, a1, a2, a3, ..., an-1]
prefix = [0, a0, a0+a1, a0+a1+a2, ..., a0+...+an-1]
```

**子数组 nums[j..i] 的和 = prefix[i+1] - prefix[j]**

## 经典应用

### 1. 除自身以外数组的乘积（两趟扫描）

不能用除法，也不能用额外 O(n) 空间。解法：**先正着扫一遍存左边乘积，再反着扫一遍乘上右边乘积**。

```cpp
vector<int> answer(n);

// 第一趟：从左到右，存左边乘积
int left = 1;
for (int i = 0; i < n; i++) {
    answer[i] = left;
    left *= nums[i];
}

// 第二趟：从右到左，乘上右边乘积
int right = 1;
for (int i = n - 1; i >= 0; i--) {
    answer[i] *= right;
    right *= nums[i];
}
```

**关键技巧**：把答案数组本身当存储中间结果的容器，省掉额外 O(n) 空间。

### 2. 和为 K 的子数组（前缀和 + 哈希表）

```cpp
unordered_map<int, int> map;   // 前缀和 → 出现次数
map[0] = 1;                    // 空前缀和为 0
int cur = 0, ans = 0;

for (int num : nums) {
    cur += num;
    int target = cur - k;      // 要找的前缀和
    if (map.count(target)) ans += map[target];
    map[cur]++;                // 记录当前前缀和
}
```

> **这个 map 是怎么来的？** 从暴力解法出发：
> 暴力：对每个 i，往回 O(i) 找子数组 → O(n²)
> 优化：用哈希表存"每个前缀和出现过几次" → O(n)

**注意：** `ans += map[target]` 不是 `ans++`。因为同一个前缀和可能在不同位置重复出现，每次出现对应一个新的合法子数组。

## 与两数之和的模板对照

| | 两数之和 | 和为 K 的子数组 |
|---|---|---|
| 遍历对象 | `nums[i]` | 前缀和 `cur` |
| 目标 | `target - nums[i]` | `cur - k` |
| hash key | 元素值 | 前缀和值 |
| hash value | 下标 | 出现次数 |
| 找到后 | `return` | `ans += 次数` |

**本质相同**：都是在问「我要找的那个值之前出现过没有？」
